Foundation

Why Typography Matters

Type isn't just words on a page. It communicates brand, hierarchy, and credibility before a single word is read.

01
Brand Signal

The fonts chosen signal the brand's personality before a visitor reads a single word. Serif vs. sans-serif, weight, and spacing all communicate luxury, trust, or energy.

02
Visual Hierarchy

Heading sizes guide the reader through the page. A broken hierarchy means the reader doesn't know where to look first — and will bounce instead of converting.

03
Readability = Trust

Body copy that's too small, too tight, or set in all caps signals poor craft. LP clients expect a premium experience. Typography that reads effortlessly is foundational.

Font Pairing Theme

  • Typically one serif and one sans-serif, or two complementary families
  • Apply consistently: headlines in one family, body in the other
  • Never exceed 2 typefaces on a site (3 max in rare cases with design justification)
  • If the pairing is already chosen by the template or design, honor it — don't swap

Do ✅

Use Consistent
Text Alignment

  • Don't center one headline and left-align the next in sections with the same layout
  • Don't mix alignment within a single section — centered headline with left-aligned body looks broken
  • Pick one and commit: left-aligned text reads more cleanly for body copy
  • Centered text is fine for hero sections or isolated callouts — not running copy

Do ✅

Minimum 14px
for Body Copy

  • 14px is the minimum for body text on desktop — not a suggestion, a floor
  • Below 14px the site may look "tight" visually but becomes inaccessible in practice
  • Small text is especially problematic on mobile — always QA at mobile scale
  • Line height of 1.5–1.7 at body size keeps paragraphs readable and airy

Don't ⛔️

Don't Set Body Copy
in All Caps

  • All caps drastically reduces readability in running text
  • All caps is for short labels, titles, nav items, or button text — never paragraphs
  • If in doubt: would you read a full article in all caps? No. Don't ask your users to either
  • Mixed casing is not a style choice here — it's a readability requirement

Core Principle

The Squint Test

"Squint at any page. You should still be able to read the hierarchy — which is the headline, which is the subhead, which is the body. If everything blurs into the same visual weight, the typography is broken."
LP Design Principles · Typography
Apply the squint test when:
  • You've finished building a page section and want to QA it
  • The client has changed heading sizes mid-build
  • You're unsure if the H2/H3 distinction is strong enough
Red flags to fix before delivery:
  • H1 and H2 feel like the same size
  • Body copy is smaller than 14px
  • Three or more typeface families in use

Your Checklist

Before You Submit
a Site for Review

Run through these three lenses on every page before QA handoff.

01
Font Pairing

Confirm only the approved 1–2 font families are in use. Check headings, body, nav, and CTAs.

  • Open inspector → check all font-family values
  • Headings use one family, body another
  • No extra families snuck in via inline styles
02
Hierarchy & Scale

Run the squint test. Every heading level should feel distinct. Body text should be comfortable at all viewport sizes.

  • H1 clearly larger than H2
  • H2 clearly larger than H3
  • Body ≥ 14px on desktop
03
Alignment & Caps

Check that text alignment is consistent within each section. Confirm no paragraphs are set in all caps.

  • Headline and body align to the same axis per section
  • No body copy is uppercase
  • All caps only on labels, nav, and CTAs
